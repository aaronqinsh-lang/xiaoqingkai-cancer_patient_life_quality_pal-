
# 小青卡 2.5 升级版需求文档 (PRD)

## 产品定义
以汝窑天青色为美学基础，整合 Gemini 实时语音与 RAG 知识库，为肿瘤患者提供全生命周期的生活质量管理与社区互助。

## 核心架构
1. **首页**：个人进度仪表盘。
2. **吾知**：结构化医学知识库，RAG 驱动。
3. **AI 助手**：核心入口，支持实时语音对话。
4. **吾享**：社区互动，采用小红书式瀑布流。
5. **吾身**：个人资料与健康档案。

## 内容创作与 SQL 规范

### 1. 文章内容结构化标准
为了确保前端 `renderStructuredContent` 能正确解析并以“温润”的 UI 呈现，所有存入 `knowledge_articles` 的内容必须遵循以下格式：

- **主标题**：使用 `## 【标题名称】` 格式。这是唯一的切割标志。
- **子标题**（可选）：使用 `### 1. 小标题` 格式。
- **正文**：常规文本，段落间空一行。
- **列表**：支持 `- 项目` 或 `1. 项目`。
- **加粗**：可以使用 `**文字**`，前端会提取文字并加粗显示，但渲染时会剥离符号。

### 2. SQL 示例模板 (Clean Content Rule)
```sql
INSERT INTO knowledge_articles (category_id, tag, title, content) 
VALUES ('category_id', '分类标签', '文章标题', 
'## 【核心结论】
这里是核心结论的正文。

## 【深度解析】
### 1. 关键点一
详细内容描述。
- 细节 A
- 细节 B

## 【温情寄语】
这里是温情寄语的正文。');
```

## 视觉与交互设计规范

### 1. 蝉翼纹理 (Cicada Wing Crackle) - 核心美学
- **设计心智**：模拟汝窑釉面的细碎开片（蟹爪纹）质感。
- **实现方式**：采用 0.3px-0.5px 的极细线条，透明度控制在 0.15-0.4 之间。

### 2. “吾享”模块主题分类
- **美食、美景、美物、美文**。

### 3. 交互细节
- **瀑布流闭环**：发布按钮移至瀑布流列表末尾。
- **渲染清洗**：前端必须对 Markdown 符号进行 100% 的清洗，严禁在 UI 中出现 `#` 或 `**`。
